# v2026.522.0

> Released: 2026-05-22

## Highlights

- **AgentDash first-run path** — Adds the `agentdash` CLI, one-line bootstrap installer, `npx create-agentdash`, frictionless `agentdash setup`, and `agentdash run` so new installs can move from checkout to a running local instance with less manual wiring.
- **CoS-led onboarding** — Rebuilds first-run onboarding around a Chief of Staff conversation, plan revision, goal materialization, agent creation, invite flows, and authenticated workspace bootstrapping.
- **Billing and launch readiness** — Adds Free/Pro limits, Stripe subscription handling, billing UI entry points, upgrade prompts, trial notices, downgrade notices, and checkout callback handling.
- **Recovery and verification loop** — Adds run-healer/self-healing recovery, stronger watchdog behavior, target-machine test automation, richer target-test diagnostics, release cadence issues, and branch-aligned release automation.

## Improvements

- **CLI and packaging** — AgentDash commands now use `agentdash` as the primary entrypoint while preserving `paperclipai` as a compatibility alias. Published package metadata now points users to the AgentDash repository, and CLI bundle generation stays aligned with all built-in adapter formatter imports.
- **macOS launchd install** — Native launchd install now writes an executable wrapper, exports env-file values before starting Node, generates missing local auth secrets, avoids checked-in machine-specific Tailscale defaults, and disables the known broken legacy plist when it points at a missing wrapper.
- **Onboarding invites** — Invite creation is wired to the real invite service, invite emails can be sent through Resend, copied invite links stay visible after send, and invite token creation paths are now consistent.
- **Agent instructions and prompts** — Worker, CEO, Chief of Staff, and generated-agent prompt surfaces now cover authentication, DoD/verdict/review behavior, and full instruction-bundle refreshes.
- **Adapters and Hermes readiness** — Improves Hermes routing/auth handling, deep-interview skill installation, unsupported CoS chat adapter handling, OpenClaw/Codex/Cursor/Gemini/OpenCode/Pi adapter coverage, and adapter environment tests.
- **UI polish** — Adds connection self-healing, offline overlays, mention typeahead fixes, billing surfaces, global error handling, improved onboarding chat layout, and clearer first-run copy.
- **CI and release operations** — PR, Docker, lockfile, target-machine, release, and Hermes audit workflows now target `main` correctly and produce more actionable diagnostics.

## Fixes

- **Security hardening** — Adds company access checks to onboarding routes, SSRF protection for assessment fetches, tiered API rate limiting, Stripe webhook secret enforcement, safer LLM output validation, and prompt-injection hardening for plan revision.
- **Billing correctness** — Handles failed invoices, upcoming trial end events, past-due tier state, invoice-paid refresh, cap-exceeded prompts, and CoS/email billing notifications.
- **Run recovery** — Fixes run-healer SQL edge cases, recovery retry ownership, stranded run handling, output watchdog decisions, and digest formatting issues.
- **Testing and reliability** — Repairs stale UI/server/e2e tests, fixes workspace-runtime behavior on dev machines with global `paperclipai`, restores typecheck/build health, and adds regression coverage for packaging drift and launchd install invariants.
- **Onboarding and authentication** — Fixes password reset flow, local-trusted bootstrap behavior, authenticated onboarding UAT, adapter-environment e2e bypasses, and sign-up workspace creation.

## Upgrade Guide

- Back up production data before upgrading. Database migrations in this release are intended to run through the normal migration path; hosted deployments should keep using a direct/session migration connection via `DATABASE_MIGRATION_URL` when runtime pooling is configured.
- Existing macOS launchd users should rerun the launchd installer after upgrading. It will generate the new AgentDash wrapper/env path and disable only the known broken legacy `com.paperclip.server.plist` case when that plist points at a missing `~/.paperclip/paperclip-launchd.sh`.
- New scripts and docs should call `agentdash`. The `paperclipai` command remains available as a compatibility alias.
- Public stable release still requires credentialed validation for external services such as Stripe and Resend before promotion.
